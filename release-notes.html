Oracle Digital Assistant Client Web SDK

Version 21.10

We are thrilled to share the 21.10 release of the Oracle Digital Assistant Native Client SDK for Web. This release introduces a couple of new features, improvements, and fixes for a few issues.

What's New

  • Feedback component - Your skill can now collect direct, real-time user feedback using a feedback component. When users complete a conversation, this component presents users with a rating gauge. If you're using the 21.10 version of the SDK, the component presents users with a star rating system; for prior versions of the SDK, the component outputs a list of action buttons. In the new 21.10 SDK, you can customize the icon used in the rating gauge, as well as its color and size. Check out the user guide for more details on the component payload and its customization.
  • Colors - The colors interface has been expanded to let you set the colors for the icons of the feedback component's rating gauge:
    • ratingStar - The color applied to the initial unselected rating stars of the feedback message.
    • ratingStarFill - The color applied to rating stars of feedback message on hover or selection.
  • icons setting - A new icons field has been exposed in the initial settings that groups all customizable icons together into a single object. This makes it easy for you to organize and configure custom icons in the widget.

    Additionally, all fields within the icons object support both image resource paths and raw SVG strings, and new icon fields have been exposed to make the widget even more customizable (logo in the following snippet is the 21.10 counterpart to the logoIcon field of prior releases, for example). The pre-21.10 icon fields are still supported, but the values passed with the icons object take precedence over those.

    Example:

    var settings = {
    URI: '<URI>',
    channelId: '<channel ID>',
    icons: {
        logo: '<svg height="24" ... ></svg>',
        rating: '<svg height="24" ... ></svg>',
        send: '../images/oda/send.png'
    }
};

    To help you migrate your existing settings with the older mechanism of passing custom icons, we have provided a helper method, mapRootIconsToNested, in the sample app that takes the current settings, maps older icon fields to new icons object, and returns the updated settings that you can save and use.

  • Latest Oracle Redwood icons - The chat widget has been updated with the latest Oracle Redwood icons. These icons indicate their functionality with a better depiction, improving the recognition from the user.

What's Improved

  • Accessibility on Edge - Improvements have been made on the HTML elements in the chat widget to make the SDK and the sample app more accessible over the Edge browser. Hidden <input> element has been made non-focusable, <title> attribute is now set on the In-Widget Webview, and appropriate CSS rules have been added to make the styling more compatible with Edge.
  • Centre-align card images - The CSS of images in cards has been updated to keep them center aligned if the width of the card increased with the widget's width.
  • Clean-up on destroy() - The SDK has been updated to better clean up its resources when the destroy() method is called by removing its stylesheet and images it uses from the web page.
  • Embedded SVG icon for launch button - Instead of using default or custom launch icon as image blob, the icon is now added as a child element in the chat widget launch button. In the case of an SVG image, this makes it much easy for the appearance of the icon to be customizable with CSS.

What's Fixed

  • Color of links in header and footer text - The colors.link theme color was not being applied to the anchor links in header and footer text. This has been fixed.
  • Distorted attachment file names - If an uploaded file's name contained Japanese characters, the file name appeared garbled in the chat widget. The SDK has been updated to properly display such file names containing Japanese and any other non-Latin characters.
  • Global styling of <hr> element - The SDK was applying styling to <hr> element which was being applied outside the chat widget as well. It has been updated to apply the styling through a class that ensures the styles are only applicable within the widget.
  • More space for custom header element - In many cases, the custom header element was being squeezed even though there was free space available in the chat header. The SDK has been updated to provide all available free space in the chat header to the custom header element if passed.
  • Triggering chatlanguagechange event on setting current language again - The SDK has been updated to prevent triggering a chatlanguagechange event if the currently set chat language is selected again, either through the UI or through the setChatLanguage() method.
